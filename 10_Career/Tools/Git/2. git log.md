# Notes on git log

## Overview
`git log` shows the commit history of a repository in reverse chronological order by default. It is used to inspect history, filter commits, study branches, and understand how a project evolved over time. [Git Documentation](https://git-scm.com/docs/git-log) [Atlassian Git Log](https://www.atlassian.com/git/tutorials/git-log)

## Basic Behavior
- `git log` lists commits reachable from the current branch tip by following parent links.
- The default output usually includes commit hash, author, date, and commit message.
- Output is shown in a pager, so you can scroll through it interactively.
- `q` exits the log view in the pager. [Git Documentation](https://git-scm.com/docs/git-log) [Git Book: Viewing the Commit History](https://git-scm.com/book/en/v2/Git-Basics-Viewing-the-Commit-History) [FreeCodeCamp Git Log](https://www.freecodecamp.org/news/git-log-command/)

## Most Useful Flags
- `--oneline`: Shows each commit in one short line with an abbreviated hash and message.
- `--graph`: Draws an ASCII branch graph.
- `--decorate`: Shows branch names, tags, and `HEAD` near commits.
- `--stat`: Shows files changed plus insertion/deletion counts.
- `-p` or `--patch`: Shows the full diff for each commit.
- `--pretty=format:"..."`: Lets you fully customize the output.
- `--abbrev-commit`: Shortens commit hashes.
- `--name-only`: Shows only file names changed in each commit.
- `--name-status`: Shows file names with change types like `A`, `M`, `D`.
- `--summary`: Gives a compact summary of file-level changes. [Git Documentation](https://git-scm.com/docs/git-log) [Atlassian Git Log](https://www.atlassian.com/git/tutorials/git-log) [FreeCodeCamp Git Log](https://www.freecodecamp.org/news/git-log-command/)

## Common Practical Commands
- `git log --oneline`
- `git log --graph --oneline --decorate`
- `git log --stat`
- `git log -p`
- `git log --pretty=format:"%h %an %s"`
- `git log -n 5`
- `git log --author="Alice"`
- `git log --grep="fix"`
- `git log --since="2026-01-01" --until="2026-04-15"`
- `git log -- file1.txt`
- `git log --all --oneline --decorate --graph` [Git Documentation](https://git-scm.com/docs/git-log) [Atlassian Git Log](https://www.atlassian.com/git/tutorials/git-log) [FreeCodeCamp Git Log](https://www.freecodecamp.org/news/git-log-command/)

## Filtering Commits
- `--author="<name>"`: Filters commits by author name or pattern.
- `--committer="<name>"`: Filters by committer.
- `--grep="<text>"`: Searches commit messages.
- `-S"<string>"`: Shows commits where the given string was added or removed in the diff.
- `-G"<regex>"`: Searches diffs using a regular expression.
- `--since` / `--after`: Shows commits after a date.
- `--until` / `--before`: Shows commits before a date.
- `-n <number>` or `--max-count=<number>`: Limits output to a fixed number of commits. [Git Documentation](https://git-scm.com/docs/git-log) [Atlassian Git Log](https://www.atlassian.com/git/tutorials/git-log) [Git Log Command Explained](https://www.freecodecamp.org/news/git-log-command/)

## Revision Range Logic
- `A..B` means commits reachable from `B` but not from `A`.
- `A...B` means the symmetric difference between `A` and `B`.
- `^commit` excludes a commit’s reachable history from the result.
- You can combine multiple starting points and exclusions in one command.
- Example: `git log main..feature` shows commits on `feature` that are not yet in `main`. [git-log Manual](https://man7.org/linux/man-pages/man1/git-log.1.html) [Git Documentation](https://git-scm.com/docs/git-log)

## Merge and Branch Cases
- `--no-merges` hides merge commits.
- `--merges` shows only merge commits.
- `--first-parent` follows only the mainline history through merges.
- `--topo-order` orders commits so parent relationships stay visually consistent.
- `--reverse` flips the order to oldest-first. [Git Documentation](https://git-scm.com/docs/git-log) [git-log Manual](https://man7.org/linux/man-pages/man1/git-log.1.html)

## Path-Specific History
- `git log -- <path>` shows commits affecting a specific file or folder.
- Without `--full-diff`, diff-based output is limited to the selected paths.
- `--full-diff` shows the full commit diff even when the history is path-limited.
- This is useful when tracing who changed a file and when. [Git Documentation](https://git-scm.com/docs/git-log)

## Special Cases
- `--all` shows commits from all refs, not just the current branch.
- `--walk-reflogs` or `-g` walks reflog history instead of normal commit ancestry.
- `--stdin` lets `git log` read revisions and paths from standard input.
- Large repositories may require filters like `-n`, `--since`, or path limits to keep output readable.
- Merge-heavy repos often become clearer with `--first-parent` or `--graph`. [Git Documentation](https://git-scm.com/docs/git-log) [git-log Manual](https://man7.org/linux/man-pages/man1/git-log.1.html)

## Pretty Format Tokens
- `%H`: Full commit hash.
- `%h`: Abbreviated commit hash.
- `%an`: Author name.
- `%ae`: Author email.
- `%ad`: Author date.
- `%cd`: Commit date.
- `%s`: Commit subject.
- `%d`: Ref names like branches and tags.
- `%p`: Parent hashes.
- `%cn`: Committer name. [Git Documentation](https://git-scm.com/docs/git-log)

## Learning Checklist
- Understand the default `git log` output.
- Learn the difference between formatting flags and filtering flags.
- Practice reading `--oneline`, `--graph`, and `--decorate` together.
- Learn how to filter by author, date, message, file, and content changes.
- Learn revision ranges: `A..B`, `A...B`, and `^commit`.
- Learn merge-related options like `--first-parent` and `--no-merges`.
- Learn how to customize output with `--pretty=format`.
- Learn path-limited history for single files and folders.
- Learn reflog history with `-g`. [Git Documentation](https://git-scm.com/docs/git-log) [Atlassian Git Log](https://www.atlassian.com/git/tutorials/git-log)

## Quick Practice Set
1. Run `git log --oneline -5`.
2. Run `git log --graph --oneline --decorate --all`.
3. Run `git log --author="your-name"`.
4. Run `git log --since="1 month ago"`.
5. Run `git log -- <file-path>`.
6. Run `git log main..feature`.
7. Run `git log --pretty=format:"%h %an %s"`.
