WinGet (Windows Package Manager, command `winget`) is Microsoft’s command‑line tool for discovering, installing, upgrading, removing, and configuring applications on Windows 10, Windows 11, and Windows Server 2025.

## What winget is

- **Winget** is the client for the Windows Package Manager service and comes as part of the App Installer component in modern Windows.
    
- It lets you work with a Microsoft‑curated software catalog plus other sources, using simple commands instead of downloading installers manually.
    

## What you can do with winget

You can think of winget as “apt/pacman for Windows apps”:

- Discover apps: `winget search <term>` to find available software in repositories.
    
- Inspect details: `winget show <id-or-name>` to see version, publisher, installer type, and supported switches.
    
- Install software: `winget install <id-or-name>` to install apps like VS Code, Git, Sourcetree, browsers, media players, etc.
    
- Upgrade software: `winget upgrade` to see what’s outdated, `winget upgrade <id>` or `winget upgrade --all` to update apps.[](https://www.youtube.com/watch?v=cODdQvF-HjM)
    
- Remove software: `winget uninstall <id-or-name>` to uninstall an app via its registered uninstaller.
    
- List inventory: `winget list` to list installed programs and versions.
    
- Manage sources: `winget source add|list|update|remove|reset` to control which repositories winget uses.
    
- Export/import setup: `winget export -o packages.json` on one PC and `winget import -i packages.json` on another to recreate your app set.
    
- Configure a machine: `winget configure <file>` to apply a full “desired state” (apps + settings) from a YAML configuration.
    
- Developer tasks: generate installer hashes (`winget hash`), validate manifests (`winget validate`), and manage pins (`winget pin`) so certain apps do not auto‑upgrade.
