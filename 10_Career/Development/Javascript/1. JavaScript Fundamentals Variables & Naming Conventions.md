# JavaScript Fundamentals: Variables & Naming Conventions

## 1. What is a Variable?
* **Definition:** Variables act as **containers** or "boxes" for storing data values.
* **Purpose:** They allow you to access, track, and modify information (like scores in a game or a user's age) throughout a program.

## 2. Declaring and Initializing Variables
* **Declaration:** Creating a variable using a keyword. 
    * *Example:* `let age;` 
    * *Note:* If accessed before a value is assigned, it returns `undefined`.
* **Assignment (=):** Using the assignment operator to give a variable a value.
    * *Example:* `age = 25;`
* **Initialization:** The process of declaring and assigning a value at the same time.
    * *Example:* `let age = 25;`

## 3. Reassignment
* Variables declared with `let` can be updated with new values.
* **Syntax:** You do not need to use the `let` keyword again when reassigning.
    * *Example:*
        ```javascript
        let points = 10;
        points = 20; // Value is now 20
        ```

## 4. Naming Rules & Guidelines
To ensure code is functional and readable, follow these requirements:

### Mandatory Rules:
* **Start Characters:** Must begin with a **letter**, an **underscore \_**, or a **dollar sign ($)**. They **cannot** start with a number.
* **Case Sensitivity:** `age` and `Age` are treated as two completely different variables.
* **Reserved Keywords:** You cannot use words that JavaScript uses for its own logic, such as `let`, `const`, `function`, or `return`.
* **No Special Characters:** Avoid symbols like `!` or `@` inside names.

### Best Practices:
* **Descriptive Names:** Use names that describe the data. Use `age` instead of a generic letter like `x`.
* **camelCase:** Start with a lowercase letter and capitalize the first letter of each subsequent word.
    * *Example:* `let playerOneScore;`
