# JavaScript Fundamentals: Template Literals & String Interpolation

## 1. What are Template Literals?
* **Definition:** A modern and flexible way to work with strings in JavaScript.
* **Syntax:** Defined using backticks ( \` ) instead of single ('') or double ("") quotes.
* **Key Benefit:** They simplify string manipulation and improve code readability, especially when dealing with variables.

## 2. String Interpolation
* **Definition:** The process of embedding variables or expressions directly into a string.
* **Syntax:** Use the `${variableName}` placeholders within the backticks.
* **Comparison:**
    * **Old Way (Concatenation):** `"My name is " + name + "!"`
    * **New Way (Interpolation):** `` `My name is ${name}!` ``

## 3. Key Features & Advantages

### A. Multiline Strings
* **Standard Strings:** Require escape characters like `\n` to create new lines.
* **Template Literals:** Naturally preserve line breaks. You can simply write the string across multiple lines in your editor.

### B. Embedding Expressions
* You are not limited to just variables; you can perform mathematical operations or call functions inside the `${ }` syntax.
* **Example:** `` `Total: ${price * quantity}` ``

### C. Enhanced Readability
* Eliminates the "quote soup" (messy mix of quotes and `+` signs) common in traditional concatenation, making the code much easier for developers to maintain.

## 4. Quick Summary Table
| Feature | Regular Strings | Template Literals |
| :--- | :--- | :--- |
| **Quote Symbol** | ' or " | ` (Backtick) |
| **Variables** | Requires `+` | `${ }` Interpolation |
| **New Lines** | Needs `\n` | Preserves naturally |
