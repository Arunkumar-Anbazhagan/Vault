# JavaScript: let vs. const vs. var

## 1. The `let` Keyword
* **Behavior:** Acts as a "flexible container."
* **Reassignment:** You can update or change the value as many times as needed.
* **Declaration:** Can be declared without an initial value (defaults to `undefined`).
* **When to use:** Use for values that change over time, such as game scores, counters, or step-by-step calculations.
    * *Example:* 
    * ```javascript
        let score = 10;
        score = 20; // Success 
        ```

## 2. The `const` Keyword
* **Behavior:** Used for "constants"—values that are intended to remain the same.
* **Reassignment:** **Cannot** be reassigned. Attempting to do so will throw a TypeError.
* **Initialization:** **Must** be assigned a value at the exact time of declaration. You cannot leave it empty.
* **When to use:** Use for configuration values, fixed settings, or any data that should not be accidentally changed.
    * *Example:* 
```javascript
        const maxScore = 100;
        maxScore = 200; // Error: Assignment to constant variable.
```
## 3. Comparison Table

| Feature | `let` | `const` |
| :--- | :--- | :--- |
| **Reassignable** | Yes | No |
| **Must Initialize** | No | Yes |
| **Purpose** | Changing data | Fixed data |

## 4. The Legacy `var` Keyword
* **Status:** Not recommended in modern JavaScript.
* **Issues:** It has a "wider scope" than `let` and `const`, which often leads to bugs and unpredictable behavior in complex programs.
