## . The Primitives

These are the building blocks you will use 90% of the time.

## **String**

Used for text, like URLs, usernames, or CSS selectors.

TypeScript

```
let browserName: string = "Chromium";
let loginUrl: string = "https://saucedemo.com";
```

## **Number**

Used for counts, timeouts, or prices. TypeScript doesn't distinguish between integers and floats; they are all `number`.

TypeScript

```
let retryCount: number = 3;
let timeoutValue: number = 30000; // 30 seconds
```

## **Boolean**

Used for `true` or `false` logic. Perfect for checking if an element is visible.

TypeScript

```
let isElementVisible: boolean = true;
let isCartEmpty: boolean = false;
```

---

## 2. Arrays

When you have a list of items (like a list of product names on a page), you use Arrays. You define them by adding `[]` after the type.

TypeScript

```
// An array of strings (e.g., product names)
let productList: string[] = ["Backpack", "Bike Light", "T-Shirt"];

// An array of numbers (e.g., price values)
let prices: number[] = [29.99, 9.99, 15.99];
```

---

## 3. Any (The "Danger" Zone)

The `any` type tells TypeScript to stop checking that variable. While it's tempting to use it when you're stuck, it defeats the whole purpose of using TypeScript.

- **Rule of Thumb:** Avoid `any` in your Playwright framework unless you are dealing with a very complex third-party library.
    

TypeScript

```
let randomData: any = "I can be anything";
randomData = 42; // No error, but dangerous!
```

---

## 4. Void

You will see `void` most often in **Functions**. It means the function does a job but doesn't "return" a value. Most Playwright actions (like clicking a button) are technically void.

TypeScript

```
function logTestStatus(status: string): void {
  console.log("Test execution status: " + status);
}
```

---

## 5. Type Inference (The Smart Feature)

TypeScript is smart. If you assign a value immediately, you don't _always_ have to write the type. It "infers" it.

TypeScript

```
let testCaseName = "Login Test"; // TS knows this is a string
// testCaseName = 123; // This would throw an error!
```
