| **Type**        | **Can assign any value to it?** | **Can access properties/methods?** | **Runtime equivalent (JS)** | **Primary Purpose**                                                                        |                                                                                      |
| --------------- | ------------------------------- | ---------------------------------- | --------------------------- | ------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------ |
| **`any`**       | **Yes**                         | **Yes** (unrestricted)             | Depends on value            | **Escape Hatch:** Disables all type checking for that variable.                            | tells compiler to skip type checking                                                 |
| **`unknown`**   | **Yes**                         | **No** (must narrow first)         | Depends on value            | **Safe Input:** For data from APIs/Users where you must verify the type before use.        | opposite to any , tells compiler , it could be anything so type check before running |
| **`void`**      | Only `undefined`                | No                                 | `undefined`                 | **No Return:** For functions that do their job and finish without returning data.          |                                                                                      |
| **`never`**     | **No**                          | No                                 | N/A (code doesn't finish)   | **Impossible State:** For functions that crash/loop, or to ensure all cases are handled.   | The `unknown` type is a type-safe counterpart of `any`.                              |
| **`null`**      | Only `null`                     | No                                 | `null`                      | **Intentional Empty:** Explicitly signaling a variable has no value.                       |                                                                                      |
| **`undefined`** | Only `undefined`                | No                                 | `undefined`                 | **Uninitialized:** The default state of a variable that hasn't been set yet.               |                                                                                      |
| **`object`**    | Any non-primitive               | Yes (if in interface)              | `Object`, `Array`, etc.     | **Non-Primitive:** Restricts the type to anything that isn't a string, number, or boolean. |                                                                                      |
