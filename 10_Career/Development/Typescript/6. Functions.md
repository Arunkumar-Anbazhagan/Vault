## 1. Normal (Function Declaration)

In a standard function declaration, types are placed immediately after the parameter name and after the closing parenthesis of the parameter list.

**Syntax:**

TypeScript

```
function functionName(parameter: parameterType): returnType {
  // logic
  return value;
}
```

**Example:**

TypeScript

```ts
function addNumbers(a: number, b: number): number {
  return a + b;
}
```

---

## 2. Arrow Function

Arrow functions are often stored in variables. You can define the types within the function definition itself, or you can define the type of the variable that holds the function.

**Syntax (Inline Typing):**

TypeScript

```ts
const functionName = (parameter: parameterType): returnType => {
  // logic
  return value;
};
```

**Example:**

TypeScript

```
const multiplyNumbers = (x: number, y: number): number => {
  return x * y;
};
```

---

## 3. Comparison Table

|**Feature**|**Normal Function**|**Arrow Function**|
|---|---|---|
|**Parameter Typing**|Inside `()`|Inside `()`|
|**Return Typing**|After `()`|After `()` but before `=>`|
|**Hoisting**|Hoisted (can call before defined)|Not Hoisted (must define first)|
|**`this` context**|Has its own `this` context|Inherits `this` from parent scope|

---

## 4. Advanced: Function Type Expressions

Sometimes you want to define the "shape" of a function separately from its implementation. This is very common when passing functions as arguments (callbacks).

TypeScript

```
// Defining the 'shape' using a Type Alias
	type MathOperation = (a: number, b: number) => number;

// Applying that shape to a function
const subtract: MathOperation = (a, b) => a - b;
```

---

### Key Takeaways

- **Optional Parameters:** Use `?` (e.g., `name?: string`).
    
- **Default Parameters:** Set a value directly (e.g., `limit: number = 10`). TS will infer the type automatically.
    
- **Void:** If the function returns nothing, use `: void`.
