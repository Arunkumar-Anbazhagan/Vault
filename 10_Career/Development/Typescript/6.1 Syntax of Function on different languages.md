|**Feature**|**TypeScript / JavaScript**|**Python**|**Java**|
|---|---|---|---|
|**Name of Concept**|Arrow Function|Lambda Expression|Lambda / Functional Interface|
|**Arrow Symbol**|`=>` (Fat Arrow)|`:` (Colon)|`->` (Thin Arrow)|
|**Variable Syntax**|`const add = (a, b) => a + b;`|`add = lambda a, b: a + b`|`BiFunction<Integer, Integer, Integer> add = (a, b) -> a + b;`|
|**Type Definition**|Inline: `(a: number): number`|Optional Type Hints: `(a: int) -> int`|Must use Interface: `Function<T, R>`|
|**Implicit Return**|Yes (if no `{ }`)|Yes (always 1 line only)|Yes (if no `{ }`)|
|**Multi-line?**|Yes, use `{ return ... }`|**No** (limited to one expression)|Yes, use `{ return ... }`|
#### 1. TypeScript (The Flexible One)

In TS, you have the most control. You can define the types exactly where the parameters are.

```Typescript
// Syntax: const Name = (params: type): returnType => logic;
const greet = (n: string): string => "Hello " + n;
```
#### 2. Python (The Minimalist)

Python uses the keyword `lambda`. It is designed to be extremely short and is almost always used for small, one-line tasks.

Python

```
# Syntax: name = lambda params: logic
greet = lambda n: "Hello " + n

# To use it:
print(greet("Alice"))
```

_Note: Python lambdas cannot have multiple lines of code or complex logic like `if/else` blocks (unless they are one-liners)._

#### 3. Java (The Strict One)

Java is the "heaviest" because it requires you to pick a **Functional Interface** (the type) before you can even write the lambda.

Java

```
// Syntax: Interface<Input, Output> name = (param) -> logic;
Function<String, String> greet = (n) -> "Hello " + n;

// To use it:
System.out.println(greet.apply("Alice"));
```

---

### Key Takeaway for Your Notes

- **TypeScript** and **Java** both use "Arrows," but the direction of the arrow points from the **Input** to the **Result**.
    
- **Python** is the "odd one out" because it uses a word (`lambda`) instead of a symbol to start the function.
    
- In all three languages, if you keep the function to **one line**, the computer is smart enough to know that line is what you want to "return."
