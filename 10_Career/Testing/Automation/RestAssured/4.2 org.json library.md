


# 🚀 THE MASTER COMPENDIUM: RESTASSURED PAYLOAD STRATEGIES & ORG.JSON

This guide covers everything from basic data passing to industry-standard patterns required for technical interviews.

---

## 1. THE BIG FOUR: COMPARISON & STRATEGY

| Method | Technical Implementation | Best For | Interview Perspective |
| :--- | :--- | :--- | :--- |
| **HashMap** | `Map<String, Object>` | Quick prototyping | "Good for simple, flat JSON; lacks type safety." |
| **org.json** | `JSONObject` / `JSONArray` | Dynamic building | "Great for manual construction without class overhead." |
| **POJO** | Java Classes + Jackson | Frameworks | "The Industry Standard. Supports Serialization/Deserialization." |
| **External File** | `new File("path.json")` | Static Data | "Best for large payloads; use placeholders for dynamicity." |

## 2. DEEP DIG: THE org.json LIBRARY

The `org.json` library (JSON-Java) is a powerful way to "build" JSON manually using code. It is more structured than a Map but doesn't require a full Java Class like a POJO.

### Core Components
* **JSONObject**: Represents `{ }`. Used for key-value pairs.
* **JSONArray**: Represents `[ ]`. Used for ordered lists.

### Practical Implementation
```java
// Step 1: Create the root object
JSONObject requestBody = new JSONObject();
requestBody.put("username", "tech_lead_2024");
requestBody.put("active", true);

// Step 2: Create a Nested Object (e.g., Profile)
JSONObject profile = new JSONObject();
profile.put("firstName", "Gemini");
profile.put("lastName", "AI");

// Step 3: Create an Array (e.g., Roles)
JSONArray roles = new JSONArray();
roles.put("admin");
roles.put("editor");

// Step 4: Assemble
requestBody.put("profile", profile);
requestBody.put("roles", roles);

// Step 5: Send with RestAssured
given()
    .contentType("application/json")
    .body(requestBody.toString()) // Convert to String
.when()
    .post("/api/users");
````

## 3. INTERVIEW "BOSS LEVEL" CONCEPTS

### A. Serialization vs. Deserialization

- **Serialization**: Java Object → JSON (Request).
    
- **Deserialization**: JSON → Java Object (Response).
    
- _Interview Tip:_ "I use Deserialization to map the response to a POJO so I can assert values using `assertEquals(user.getName(), "John")` instead of messy JsonPaths."
    

### B. The Dependency Layer

To make HashMaps or POJOs work, you **must** mention these in your `pom.xml`:

- **Jackson Databind** (Most common) or **Gson**.
    
- Without these, RestAssured will throw an error: `IllegalStateException: Cannot serialize object because no JSON serializer found`.
    

### C. The "Lombok" Advantage

In interviews, mention **Project Lombok**.

- Instead of writing 50 lines of Getters/Setters for a POJO, you use `@Data`. It shows you know how to write clean, modern code.
    

### D. Handling Dynamic Data in Static Files

- _Question:_ "If you use an external JSON file, how do you change the 'email' for every test?"
    
- _Answer:_ "I read the file into a String, use `.replace("{{email}}", "new@mail.com")`, and then pass that string into the `.body()` method."

## 5. TOP 3 INTERVIEW QUESTIONS TO PREPARE

1. **"Why choose POJO over HashMap?"** (Answer: Type safety, readability, and ability to deserialize responses for complex assertions).
    
2. **"What happens if the API adds a new field you didn't define in your POJO?"** (Answer: Use `@JsonIgnoreProperties(ignoreUnknown = true)` to prevent the test from crashing).
    
3. **"How do you send an array of objects?"** (Answer: Create a `List<POJOClass>` or a `JSONArray` and add multiple `JSONObject` instances to it).
