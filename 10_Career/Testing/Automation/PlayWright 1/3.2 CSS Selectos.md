The session explains how Playwright supports generic locators like CSS and XPath, though Playwright-specific locators are prioritized. The tutorial moves from basic CSS syntax (ID, Class, Attributes) to advanced techniques like pseudo-classes (`:nth-child`), pattern matching (starts-with, ends-with, contains), and logical negation (`:not`).

---

### **Detailed Session Notes**

#### **1. Introduction to Locators & Priority**

- **Locator Priority in Playwright:**
    
    1. Playwright-specific locators (Built-in).
        
    2. **CSS Locators** (Preferred over XPath for speed). [00:35](http://www.youtube.com/watch?v=FClj0-oGP-w&t=35)
        
    3. XPath Locators (Recommended only if others fail).
        
- **CSS Definition:** Cascading Style Sheets are used for styling web pages, but in automation, they serve as a "Locator Type" to find elements based on their attributes.[01:32](http://www.youtube.com/watch?v=FClj0-oGP-w&t=92)]
    

#### **2. Generic CSS Locator Combinations**

The instructor outlines four primary ways to form a relative CSS locator: [[06:05](http://www.youtube.com/watch?v=FClj0-oGP-w&t=365)]

- **Tag and ID:** Uses the `#` symbol.
    
    - _Syntax:_ `tagname#idvalue` or simply `#idvalue`.
        
- **Tag and Class:** Uses the `.` symbol. [[07:46](http://www.youtube.com/watch?v=FClj0-oGP-w&t=466)]
    
    - _Syntax:_ `tagname.classname` or simply `.classname`.
        
- **Tag and Attribute:** Uses square brackets `[]`. [[08:17](http://www.youtube.com/watch?v=FClj0-oGP-w&t=497)]
    
    - _Syntax:_ `tagname[attribute='value']`.
        
- **Tag, Class, and Attribute:** Combines all for higher specificity. [[09:16](http://www.youtube.com/watch?v=FClj0-oGP-w&t=556)]
    
    - _Example:_ `input.search-box[name='q']`.
        

#### **3. Absolute vs. Relative CSS Locators**

- **Absolute CSS:** Starts from the root `html` tag and uses the `>` (greater than) symbol for navigation. [[41:04](http://www.youtube.com/watch?v=FClj0-oGP-w&t=2464)]
    
    - _Example:_ `html > body > div > h1`.
        
    - _Disadvantage:_ Extremely brittle; if the page structure changes slightly, the locator breaks. [[59:20](http://www.youtube.com/watch?v=FClj0-oGP-w&t=3560)]
        
- **Relative CSS:** Jumps directly to the element using unique attributes. This is the standard practice in automation. [[01:00:23](http://www.youtube.com/watch?v=FClj0-oGP-w&t=3623)]
    

#### **4. Advanced Navigation & Pseudo-Classes**

Since CSS only supports **top-to-bottom** navigation (unlike XPath's bi-directional capabilities), specific methods are used to find children: [[46:41](http://www.youtube.com/watch?v=FClj0-oGP-w&t=2801)]

- **`:first-child`**: Selects the first child element of a parent. [[01:07:48](http://www.youtube.com/watch?v=FClj0-oGP-w&t=4068)]
    
- **`:last-child`**: Selects the last child element. [[01:08:23](http://www.youtube.com/watch?v=FClj0-oGP-w&t=4103)]
    
- **`:nth-child(n)`**: Selects the element at a specific index (1-based). [[01:08:50](http://www.youtube.com/watch?v=FClj0-oGP-w&t=4130)]
    

#### **5. Pattern Matching Operators**

Useful when attribute values are dynamic or very long: [[01:10:54](http://www.youtube.com/watch?v=FClj0-oGP-w&t=4254)]

- **`^=` (Starts-with):** `[class^='MA']` matches "Main". [[01:11:32](http://www.youtube.com/watch?v=FClj0-oGP-w&t=4292)]
    
- **`$=` (Ends-with):** `[id$='2']` matches "para2". [[01:12:38](http://www.youtube.com/watch?v=FClj0-oGP-w&t=4358)]
    
- **`*=` (Contains):** `[class*='ai']` matches "Main". [[01:14:50](http://www.youtube.com/watch?v=FClj0-oGP-w&t=4490)]
    

#### **6. Logical Operators and Sibling Navigation**

- **`:not()` (Negation):** Used to exclude elements with certain properties. [[01:18:01](http://www.youtube.com/watch?v=FClj0-oGP-w&t=4681)]
    
    - _Example:_ `p:not([class='main'])` finds all paragraphs except those with the class 'main'.
        
- **`+` (Immediate Sibling):** Targets the element directly following another element at the same level. [[01:23:23](http://www.youtube.com/watch?v=FClj0-oGP-w&t=5003)]
    

#### **7. Practical Implementation in Playwright**

To use these in code, the `page.locator()` method is employed: [[11:40](http://www.youtube.com/watch?v=FClj0-oGP-w&t=700)]

TypeScript

```
// Example using Tag and ID
await page.locator('input#small-searchterms').fill('t-shirts');

// Example verifying visibility before action
const searchBox = page.locator('#small-searchterms');
await expect(searchBox).toBeVisible(); [00:21:29]
```
