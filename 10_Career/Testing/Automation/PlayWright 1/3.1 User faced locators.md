## 🎭 Playwright Locators: User-Facing

Playwright recommends using locators that resemble how a **real human** perceives a webpage. This makes tests more resilient to changes in the HTML code.

### 1. `page.get_by_role()`

This is the top-tier recommendation. It locates elements by their **ARIA role** (like button, checkbox, heading) and their accessible name.

- **Why use it?** It ensures your app is accessible to screen readers while making your tests robust.
    
- **Example:** `page.get_by_role("button", name="Submit")`
    
- **Reference:** [Playwright Role Locators](https://www.google.com/search?q=https://playwright.dev/docs/locators%23locate-by-role)
    

### 2. `page.get_by_label()`

Used for form fields. It finds the input element associated with a specific `<label>` text.

- **Example:** `page.get_by_label("Username")`
    
- **Reference:** [Playwright Label Locators](https://www.google.com/search?q=https://playwright.dev/docs/locators%23locate-by-label)
    

### 3. `page.get_by_text()`

Finds an element containing specific text. It’s great for non-interactive elements like paragraphs or spans.

- **Example:** `page.get_by_text("Welcome back!")`
    

### 4. `page.get_by_placeholder()`

Locates input fields by the hint text displayed inside them before a user types.

- **Example:** `page.get_by_placeholder("name@example.com")`

## 🔍 Understanding Selectors

While **Locators** are the "how" (the API you call), **Selectors** are the "what" (the string engine used to point to the element).

When user-facing locators aren't enough, we use specific selector engines:

|**Selector Type**|**Description**|**Example**|
|---|---|---|
|**CSS**|Uses standard CSS syntax. Fast and common.|`page.locator("button.submit-btn")`|
|**XPath**|Navigates the XML/HTML tree structure.|`page.locator("//div[@id='root']//input")`|
|**Attribute**|Targets specific HTML attributes.|`page.locator("[data-testid='login-button']")`|
