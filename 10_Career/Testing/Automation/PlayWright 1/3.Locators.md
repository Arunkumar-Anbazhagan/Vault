### 1.[[3.1 User faced locators]]  (Recommended) 👤

These are the "gold standard" because they find elements the same way a human does. If a developer changes the code but the button still says "Submit," these locators won't break.

These are the recommended built-in locators.

- [page.getByRole()](https://playwright.dev/docs/locators#locate-by-role) to locate by explicit and implicit accessibility attributes.
- [page.getByText()](https://playwright.dev/docs/locators#locate-by-text) to locate by text content.
- [page.getByLabel()](https://playwright.dev/docs/locators#locate-by-label) to locate a form control by associated label's text.
- [page.getByPlaceholder()](https://playwright.dev/docs/locators#locate-by-placeholder) to locate an input by placeholder.
- [page.getByAltText()](https://playwright.dev/docs/locators#locate-by-alt-text) to locate an element, usually image, by its text alternative.
- [page.getByTitle()](https://playwright.dev/docs/locators#locate-by-title) to locate an element by its title attribute.

### 2. Developer-Focused Locators 🏗️

Sometimes the UI doesn't have clear labels or roles. In these cases, we use attributes specifically meant for testing or identifying elements in the code.

- **`getByTestId()`**: Uses a special attribute (usually `data-testid`) that developers add specifically for automation.
    
- [[3.2 CSS Selectos]] Uses standard web styling syntax (e.g., `.class-name` or `#id-name`).
    
- **XPath**: Uses the "path" or coordinates of the element within the HTML tree.
    

### 3. Combined & Filtered Locators 🖇️

Playwright allows you to chain locators together to be very precise.

- **Filtering**: You can take a list of buttons and say "find the one that has the text 'Save'".
    
- **Nesting**: You can find a specific `div` first, and then look for a button _inside_ only that `div`.
