# [What is a Request Body in an API?](https://academy.apidog.com/what-is-a-request-body-in-an-api-1807844m0)

## Overview
The **Request Body** is the part of an HTTP request used to send structured data to a server. Unlike parameters that reside in the URL, the body is contained within the request itself and is primarily used for creating or updating resources.

---

## 1. When to Use a Request Body
Request bodies are typically associated with specific HTTP methods:
* **POST:** To create a new resource.
* **PUT:** To replace an existing resource entirely.
* **PATCH:** To update specific parts of a resource.
* *Note:* **GET** requests generally do not include a body.

---

## 2. Common Data Formats
While modern APIs favor **JSON**, several formats are supported:
* **JSON (application/json):** The industry standard for structured data (objects and arrays).
* **XML (application/xml):** Often found in legacy or enterprise systems.
* **Multipart/Form-Data:** Essential for **file uploads** (images, PDFs, etc.) or mixed content.
* **URL-Encoded:** Used for simple form field submissions.

---

## 3. Understanding the Schema
Before sending a request body, you must follow the defined **Schema**, which dictates:
* **Fields:** Which keys are available (e.g., `id`, `name`, `status`).
* **Data Types:** Whether a value should be a string, integer, boolean, or array.
* **Constraints:** Required vs. optional fields, and allowed values (Enums).

---

## 4. Key Takeaways
* **Content-Type Header:** Always set this to match your body format (e.g., `application/json`) so the server knows how to parse the data.
* **Structure:** Bodies can handle complex, nested data that query parameters cannot.
* **Tooling:** Platforms like [Apidog](https://apidog.com/) can auto-generate request bodies based on the provided API schema to ensure accuracy.
