## Concept:
To use Claude Code with mobile apps through channels, you install a pre-built channel plugin for your messaging platform, like Telegram, Discord, or iMessage. The plugin runs locally on your computer and polls the platform's API for new messages. When someone sends you a message on the mobile app, the plugin receives it and forwards it to Claude Code, which can then read and respond. You configure the plugin in your MCP config file, and Claude Code spawns it automatically. The plugin acts as a bridge between the mobile app and Claude Code, so you can chat with Claude through your phone without exposing any URLs to the internet.

|Term|What it means|
|---|---|
|**Channel plugin**|A pre-built connector for a specific messaging app|
|**Polls the API**|The plugin keeps asking the platform "any new messages?" at regular intervals|
|**MCP config file**|The config file where you register which plugins Claude Code should load|
|**Claude Code spawns it**|Claude Code automatically starts the plugin when it launches — no manual steps|
## Why "No URLs Exposed"?

This is the key security benefit. The plugin initiates the connection _outward_ to Telegram/Discord's API (like a client), rather than opening a port on your machine to receive incoming connections. So your computer is never publicly accessible on the internet — it's a one-way outbound connection, much safer than running a webhook server.[](https://roadmap.sh/claude-code)

Since you're already familiar with API polling concepts from your API testing work, this pattern should feel intuitive — it's essentially the same as polling a REST endpoint, just done by the plugin automatically in the background.

## Prerequisites (All Channels)

Before configuring any channel, ensure you have:[](https://code.claude.com/docs/en/channels)

- Claude Code **v2.1.80 or later** installed and authenticated with a **claude.ai** account
    
- **[Bun](https://bun.sh/)** installed (verify with `bun --version`)
    
- Team/Enterprise users: admin must enable channels via `channelsEnabled: true`

## Fakechat (Demo / Quickstart)

1. **Install the plugin** — In a Claude Code session, run:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `/plugin install fakechat@claude-plugins-official`
    
2. **Restart with channel enabled**:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `claude --channels plugin:fakechat@claude-plugins-official`
    
3. **Push a test message** — Open `http://localhost:8787` and type any message.[](https://code.claude.com/docs/en/channels)

## Telegram

1. **Create a bot** — Open [BotFather](https://t.me/BotFather) in Telegram, send `/newbot`, give it a display name and username ending in `bot`, then copy the returned token.[](https://code.claude.com/docs/en/channels)
    
2. **Install the plugin**:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `/plugin install telegram@claude-plugins-official`
    
    If not found, run `/plugin marketplace update claude-plugins-official` first, then retry.
    
3. **Reload plugins**:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `/reload-plugins`
    
4. **Configure your token**:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `/telegram:configure <YOUR_BOT_TOKEN>`
    
    This saves to `~/.claude/channels/telegram/.env`.
    
5. **Restart with channels enabled**:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `claude --channels plugin:telegram@claude-plugins-official`
    
6. **Pair your account** — Send any message to your bot in Telegram; it replies with a pairing code. Back in Claude Code, run:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `` /telegram:access pair`<PAIRING_CODE>` ``
    
7. **Lock down access** to allowlist only:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `/telegram:access policy allowlist`

## Discord

Steps follow the same pattern as Telegram:[](https://code.claude.com/docs/en/channels)

1. Install: `/plugin install discord@claude-plugins-official`
    
2. Reload: `/reload-plugins`
    
3. Configure with your Discord bot token via the `/discord:configure` command
    
4. Restart: `claude --channels plugin:discord@claude-plugins-official`
    
5. Send a message to your Discord bot → get a pairing code → approve it in Claude Code

## iMessage (macOS only)

1. **Install the plugin**:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `/plugin install imessage@claude-plugins-official`
    
2. **Restart with channels enabled**:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `claude --channels plugin:imessage@claude-plugins-official`
    
3. **Pair** — Texting yourself bypasses the gate automatically. To add other contacts, use:[](https://code.claude.com/docs/en/channels)
    
    text
    
    `/imessage:access allow <handle>`

## Enterprise Controls (Team/Enterprise Admins Only)

1. **Enable channels** — Go to `claude.ai → Admin settings → Claude Code → Channels` and toggle `channelsEnabled: true`.[](https://code.claude.com/docs/en/channels)
    
2. **Restrict plugins** (optional) — Set `allowedChannelPlugins` in managed settings to control which plugins can register:[](https://code.claude.com/docs/en/channels)
    
    ```json
    
    {   "channelsEnabled": true,  "allowedChannelPlugins": [    { "marketplace": "claude-plugins-official", "plugin": "telegram" },    { "marketplace": "claude-plugins-official", "plugin": "discord" }  ] }
    
    ```
