# 1. Skills: 
To understand **"Skills"** within the context of **Claude Code**, it is essential to first understand what Claude Code actually is: it is a command-line interface (CLI) tool that allows you to interact with Claude directly in your terminal to build, debug, and manage codebases.

When we talk about "Skills" in this environment, we are referring to the **tool-use capabilities** that allow Claude to move beyond just "writing text" and actually perform "actions" on your local machine.

Here is a breakdown of what Skills mean in the Claude Code ecosystem:

### 1. The Core Concept: From Text to Action

Normally, an AI model is limited to generating text. In the context of Claude Code, "Skills" are the **functions and APIs** exposed to the model that enable it to interact with your operating system and development environment.

Without these skills, Claude would just be a chatbot. With these skills, Claude becomes an **agent**.

### 2. Key Categories of Skills

Claude Code is equipped with several "skills" (or tools) that it can invoke autonomously based on the tasks you give it:

- **File System Operations:** Claude has the skill to read, write, create, delete, and list files. This is fundamental for refactoring code, creating new features, or fixing bugs across multiple files.
- **Command Execution:** Claude can run terminal commands (like `npm install`, `pytest`, `git commit`, or `grep`). This allows it to install dependencies, run tests, and check the status of your repository.
- **Code Search/Navigation:** Claude uses techniques like `grep` or specific file-indexing skills to "understand" the context of your project before it makes a change.
- **Goal-Oriented Reasoning:** This is a "meta-skill." Claude doesn't just execute a command; it plans. If you ask it to "fix the bug in the auth flow," it will use its skills to search for the relevant files, read the code, hypothesize the fix, write the code, and run tests to verify.

### 3. Why "Skills" Matter (The Difference)

To understand the _difference_ between Claude Code and a standard Chat UI (like ChatGPT or the standard Claude web interface), you have to look at the **execution loop**:

- **Standard AI (No Skills/Limited):** You describe a problem -> AI writes code snippet -> You copy-paste and test it yourself.
- **Claude Code (With Skills):** You describe a problem -> Claude analyzes your directory -> Claude executes a test -> Claude identifies the failure -> Claude modifies the code -> Claude runs the test again -> **Claude reports the success.**

### 4. Safety and Control

Because Claude Code has "skills" that can modify your files and run shell commands, it operates within a **sandbox-like philosophy**. It is designed to be transparent:

- It logs every action it takes.
- It asks for permission (if configured) or follows your guidance for major changes.
- It is designed to work _with_ you, using its skills to reduce the "drudge work" of coding, while you remain the architect.

### Summary

In the roadmap for Claude Code, **Skills** represent the bridge between **intelligence** (the model's reasoning) and **utility** (the ability to manipulate your computer). By mastering how to prompt Claude Code, you are essentially learning how to orchestrate these skills to automate your development workflow.

# 2. Context:

A context window is the total amount of information a model can hold in its active memory at one time, including your conversation history, the files it has read, and the results of any terminal commands it has executed. It functions like a temporary workspace where every word and symbol (counted as tokens) fills up a limited capacity. As this window fills, Claude Code may begin to "forget" earlier details or become less accurate, which is why managing context through commands like `/compact` is essential to keep responses sharp and costs under control.
