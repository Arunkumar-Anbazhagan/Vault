### Commands Used for Wi-Fi Scan Behavior Testing (Android via ADB)

---

#### 1. Check Background Scanning Status
```bash
adb shell settings get global wifi_scan_always_enabled
```

Description:
Retrieves whether background Wi-Fi scanning is enabled.

Usage:
Run before testing to confirm scan behavior configuration.

Expected Output:
- 1 → Background scanning enabled  
- 0 → Background scanning disabled  

---

#### 2. Enable Background Wi-Fi Scanning
```bash
adb shell settings put global wifi_scan_always_enabled 1
```

Description:
Allows Wi-Fi scanning even when Wi-Fi is turned OFF.

Usage:
Used to validate that scan results are still available when Wi-Fi is disabled.

---

#### 3. Disable Background Wi-Fi Scanning
```bash
adb shell settings put global wifi_scan_always_enabled 0
```


Description:
Stops background Wi-Fi scanning when Wi-Fi is OFF.

Usage:
Used to verify behavior when scanning is completely disabled.

#### Retrieve Full Wi-Fi Service Dump
```bash
adb shell dumpsys wifi
```


Description:
Provides detailed Wi-Fi system information including scan results.

Usage:
Used to inspect SSID, BSSID, signal strength, and scan history.

---

#### 5. Filter SSID (Windows PowerShell)
```bash
adb shell dumpsys wifi | findstr /i "SSID"
```

Description:
Filters output to display only SSID-related lines.

Usage:
Quick check for available networks without parsing full dump.

---

#### 6. Filter SSID + BSSID (Linux / Android Terminal)
```bash
adb shell dumpsys wifi | grep -i "SSID\|BSSID"

```
Description:
Extracts SSID and BSSID from scan results.

Usage:
Used for focused validation of network identifiers.

---

#### 7. Validate Scan Behavior Over Time
```bash
adb shell dumpsys wifi | findstr /i "SSID"
```

(re-run after 10–15 seconds)

Description:
Repeated execution to observe changes in scan results.

Usage:
- Changing values → real-time scanning  
- Same values → cached results  

---

### Recommended Test Flow

1. Check scan setting  
2. Enable/disable background scanning  
3. Toggle Wi-Fi ON/OFF  
4. Run scan commands  
5. Repeat after delay  
6. Compare outputs  

---

### Key Observation

Wi-Fi scan results may still be visible even when Wi-Fi is OFF if background scanning is enabled.
